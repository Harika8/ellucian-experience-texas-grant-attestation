How this maps to the real flow

  1. Experience card loads → calls a Data Connect GET pipeline.
  2. Pipeline resolves the student via Ethos persons, checks eligibility with award-maintenance (RPRAWRD) and reads any existing row from applicant-requirements (RRAAREQ).
  3. Card shows the form only if eligible and no row yet.
  4. On submit → POST pipeline writes the answer to Banner RRAAREQ via the applicant_requirements Business API.
  5. Next load: a row exists → card renders nothing (no re-submission).
